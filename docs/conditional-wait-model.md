# 院內條件輪候模型

## 預測目標

「15 分鐘後醫院公開 p50」與「病人再等 15 分鐘仍未見醫生後的剩餘時間」不是同一目標。

個人院內輪候目標是 P(T - t <= r | T > t, 分流、同級位置、高優先度隊列、醫院狀態歷史)。T 是首次見醫生時間，t 是已等待時間；T > t 表示刷新時仍未見醫生。這是條件存活／競爭優先隊列問題，不能用官方到院估算減去 t。

## 可觀察與不可識別部分

- 官方每 15 分鐘提供到院時院級 p50/p95，以及是否正處理危殆／危急或多宗復甦個案。
- 使用者可提供已等待時間、同級位置與較高優先度個案壓力。
- API 沒有逐分流到達數 λ(t)、首次見醫生／完成數 μ(t)、逐更人手或病人級 T。
- 因此目前可更新淨隊列狀態，但不能由公開資料單獨分解需求與容量，也不能校準個人 hazard。

## 07:00 外部先驗

病人現場觀察與醫院工作人員反饋指約 07:00 人手及效率較高；另有 09:00 前工作人口自行到院或陪同家人、令到達增加的假設。兩者可能同時成立，公開等候曲線只顯示淨結果。

檢驗應保留未參與模型選擇的日期，檢查控制醫院、星期、季節、當前 p50/p95、趨勢與危急訊號後，06–08 的預測殘差是否仍有結構轉折。即使等候在 07:00 為低位，也不能直接解讀成人員效率。

## 後續可校準模型

取得匿名首次見醫生事件，或至少逐院、逐 15 分鐘、按分流級別的到達與完成數後，可建立：

1. 分流優先的離散事件模擬；
2. 隨時間變化的需求 λ_h,c(t) 與容量 μ_h(t)；
3. 以每次仍未服務作 landmark 更新的條件存活模型；
4. 與 persistence 和現有歷史類比模型作封鎖時間回測及機率校準。

在取得這些標籤前，網站只呈現條件隊列狀態，不輸出看似精確的個人倒數。
